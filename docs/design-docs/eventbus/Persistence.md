# EventBus persistence

EventBus persistence defines the PostgreSQL contract for durable routing
configuration and weak TargetSession runtime state. It covers Event Routing Rule
configuration, TargetSession rows, inbound side-channel entries, repair, and
runtime cleanup.

PostgreSQL remains the source of committed business facts for BullX, but the
TargetSession runtime tables in this document are not business truth. They are
weak runtime state used by [EventBus core](./Core.md) to hand accepted Events to
long-running TargetSession jobs.

## Data model

PostgreSQL stores durable EventBus configuration and weak TargetSession runtime
state. UUID primary keys are generated by BullX with `BullX.Ecto.UUIDv7` and
`BullX.Ext.gen_uuid_v7/0`. Closed sets use PostgreSQL native enum types.

Event Routing Rules are durable logged configuration. TargetSession state and
inbound side-channel entries use PostgreSQL `UNLOGGED` tables. These runtime
records can survive ordinary BEAM process crashes, but PostgreSQL may truncate
them after a PostgreSQL crash or unclean shutdown. This tradeoff is acceptable
because these records are not business facts.

Unless a later implementation proves a stronger need, runtime tables must not
create durable foreign-key assumptions from UNLOGGED runtime tables to logged
business or configuration tables. Runtime code validates referenced IDs at
runtime.

## `event_routing_rules`

`event_routing_rules` is a durable logged table for database-owned routing
configuration. Code-owned built-in system command routes are not persisted in
this table; `RoutingTable` merges them into the runtime snapshot from code.

| Field | Type | Requirement |
| --- | --- | --- |
| `id` | `uuid` | Primary key, BullX-side UUIDv7 |
| `name` | `text` | Not null; unique durable canonical name |
| `active` | `boolean` | Not null, default `true` |
| `priority` | `integer` | Not null |
| `match_expr` | `text` | Not null |
| `target_type` | `eventbus_target_type` | Not null |
| `target_ref` | `text` | Null only for Blackhole |
| `scope_fields` | `text[]` | Not null |
| `window_type` | `target_session_window_type` | Not null |
| `window_ttl_seconds` | `integer` | Required only for `rolling_ttl` |
| `inserted_at` | `timestamptz` | Not null |
| `updated_at` | `timestamptz` | Not null |

Constraints and indexes:

- Unique `name` across all rows. `name` is the durable idempotency key used by
  `BullX.EventBus.RuleWriter.upsert_by_name/2`; it is not merely a display
  label.
- Check `name = btrim(name)` and `name <> ''`.
- Unique `priority` across all rows.
- Check `priority > 0`. Positive priorities are reserved for database-owned
  rules. Code-owned built-in routes may use reserved negative priorities in the
  runtime snapshot because they are not PostgreSQL rows.
- Check `target_ref IS NULL` when `target_type = 'blackhole'`.
- Check `target_ref IS NOT NULL` when `target_type <> 'blackhole'`.
- Check `target_ref = btrim(target_ref)` and `target_ref <> ''` when
  `target_type <> 'blackhole'`.
- Check `window_ttl_seconds IS NOT NULL AND window_ttl_seconds > 0` when
  `window_type = 'rolling_ttl'`.
- Check `window_ttl_seconds IS NULL` when `window_type = 'new_per_event'`.
- The rule writer stores `scope_fields = []`, `window_type = 'new_per_event'`,
  and `window_ttl_seconds = NULL` when `target_type = 'blackhole'`.

Rule semantics and matcher behavior are defined in
[EventBus matcher](./Matcher.md).

## `target_sessions`

`target_sessions` is an `UNLOGGED` runtime table.

| Field | Type | Requirement |
| --- | --- | --- |
| `id` | `uuid` | Primary key, BullX-side UUIDv7 |
| `event_routing_rule_id` | `uuid` | Not null |
| `target_type` | `eventbus_target_type` | Not null |
| `target_ref` | `text` | Not null |
| `scope_key` | `text` | Not null |
| `window_key` | `text` | Not null |
| `status` | `target_session_status` | Not null |
| `oban_job_id` | `bigint` | Null until associated |
| `last_processed_entry_seq` | `bigint` | Not null, default `0` |
| `expires_at` | `timestamptz` | Null except for rolling/window expiry |
| `terminal_reason` | `text` | Null or safe terminal diagnostics |
| `inserted_at` | `timestamptz` | Not null |
| `updated_at` | `timestamptz` | Not null |

`event_routing_rule_id` stores the matched route id from the runtime route-table
snapshot. It may identify either a PostgreSQL `event_routing_rules.id` or a
stable code-owned built-in route id. The column is not a foreign key;
TargetSession runtime rows must remain able to reference code-owned routes that
are not persisted in PostgreSQL.

Constraints and indexes:

- Partial unique index on
  `(event_routing_rule_id, target_type, target_ref, scope_key, window_key)` where
  `status = 'active'`.
- Index on `expires_at` for cleanup and expiry.
- Index on `oban_job_id`.

The hard max runtime deadline is computed as `inserted_at + 24 hours`. Queries
that need to find TargetSessions near the hard cap compute against
`inserted_at`; this invariant does not require a separate deadline column.

`oban_job_id` is the runtime association between the TargetSession and its
long-running Oban job. The job ensure path updates this field idempotently while
holding the `target_sessions` row lock, and it uses Oban unique job
configuration to keep at most one non-terminal job for a `target_session_id`.

Blackhole never creates TargetSessions.

`target_ref` is an opaque stable Target reference. UUID-backed Targets store the
canonical UUID string. Code-owned Targets, such as Command Target handlers, store
stable registry ids such as `bullx.system.status`. EventBus runtime code must
not derive module names from this value.

## `target_session_entries`

`target_session_entries` is an `UNLOGGED` inbound side-channel runtime table.

| Field | Type | Requirement |
| --- | --- | --- |
| `id` | `uuid` | Primary key, BullX-side UUIDv7 |
| `entry_seq` | `bigserial` / `bigint` | Not null ordering column |
| `target_session_id` | `uuid` | Not null |
| `event_source` | `text` | Not null |
| `event_id` | `text` | Not null |
| `dedupe_hash` | `text` | Not null, internal idempotency key |
| `cloud_event` | `jsonb` | Not null |
| `routing_context` | `jsonb` | Not null |
| `appended_at` | `timestamptz` | Not null |

Constraints and indexes:

- Unique `dedupe_hash`.
- Index on `(target_session_id, entry_seq)`.

`cloud_event` and `routing_context` are runtime handoff records. They are not
business facts, audit truth, or replay truth.

## Repair and runtime cleanup

`target_sessions` and `target_session_entries` are `UNLOGGED` runtime state.
After a PostgreSQL crash or unclean shutdown, PostgreSQL may truncate them.

Oban jobs are durable logged records, so an Oban job may reference a
`target_session_id` whose runtime row no longer exists. The TargetSession worker
must handle a missing `target_session_id` safely by discarding or completing the
orphaned job without inventing business facts.

Application boot repair prunes or completes Oban jobs whose TargetSession
runtime row is missing. It also idempotently ensures the associated long-running
Oban job for every `active` TargetSession runtime row. This repair is safe
because EventBus and TargetSession runtime records are not business facts.

Terminal `target_sessions` and their `target_session_entries` are retained only
for bounded diagnostics. Cleanup deletes `closed`, `failed`, and `expired`
sessions after configurable `target_session_runtime_retention_seconds`, which
must have a default. Unless the implementation adds a clearer terminal
timestamp, cleanup uses `updated_at` as the terminal timestamp.

Cleanup does not directly delete active sessions. If an active session has
exceeded `inserted_at + 24 hours` or its `expires_at` runtime window has passed,
cleanup first marks it `expired`, then applies terminal retention.

Streaming output retention and Redis cleanup belong to
[TargetSession streaming output](./StreamingOutput.md).
