# Elixir Skill

This directory contains the Elixir coding skill used by repository agents.

`SKILL.md` is the current entrypoint. Supporting reference files in this
directory provide deeper guidance for language patterns, OTP, Ecto, testing,
documentation, type specs, and production concerns.

BullX-specific rules still come from the repository guidance in `AGENTS.md`.
When generic skill guidance conflicts with BullX repo guidance, follow the repo
guidance. In particular, BullX module documentation should describe
non-obvious contracts and invariants; it should not be added just to restate
module names, fields, callbacks, or standard Elixir behavior.
